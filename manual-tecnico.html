ejecutivo.

Sistema automático que genera manuales de bienvenida multilingüe para propiedades 3villas extrayendo datos de Caspio. Un fichero HTML estático en GitHub Pages, con un proxy seguro en Cloudflare Workers.

github.com/jordise/

⚡ Proxy (Worker)

dash.cloudflare.com
caspio-proxy.jordi-89b.workers.dev

🗄️ Base de datos

threevillas.caspio.com
Tabla: TaVillas · Campo clave: Hostawayid

Arquitectura
del sistema.

Sin servidor propio. Todo funciona desde un fichero HTML estático publicado en GitHub Pages, con un proxy serverless en Cloudflare Workers que gestiona la autenticación con Caspio.

01
Navegador
index.html en GitHub Pages. Usuario introduce Hostawayid.
02
CF Worker
Proxy en Cloudflare. Gestiona OAuth internamente. Sin exponer credenciales.
03
Caspio REST
c1abn432.caspio.com. Devuelve el registro de TaVillas.
04
Google Drive
Thumbnail de la carpeta del campo Drive.
05
Manual HTML
8 páginas × 5 idiomas. Listo para imprimir en PDF.

Consulta Worker → Caspio

GET ?action=data&table=TaVillas&id=92737
→ Worker obtiene token OAuth interno
→ GET /records?q.where=Hostawayid=92737
→ Devuelve JSON con todos los campos
→ mapRecord() extrae los campos necesarios
→ renderLang() × 5 idiomas → Manual HTML

Foto de portada

Campo Drive en TaVillas:
drive.google.com/drive/folders/ID

→ Regex extrae el folder ID
→ drive.google.com/thumbnail?id=ID&sz=w900
→ Se usa como src del img de portada
→ Si falla: onerror oculta el img

Componentes
del sistema.

GitHub Pages

Hospeda el index.html. El único archivo del repositorio. Repositorio público.

>jordise.github.io/3villas-manuals
>github.com/jordise/3villas-manuals
Para actualizar el programa:
  1. Pide el nuevo index.html a Claude
  2. Ve al repositorio de GitHub
  3. Sube el archivo nuevo (arrastra o edita)
  4. "Commit changes" · Espera 2 min · Ctrl+Shift+R
⚠ Repo público: Nunca pongas credenciales en el index.html. Son visibles para todos.

Cloudflare Worker

Proxy entre el navegador y Caspio. Gestiona la autenticación OAuth sin exponer credenciales.

caspio-proxy.jordi-89b.workers.dev

>dash.cloudflare.com
Para editar el código del Worker:
  1. dash.cloudflare.com → Workers y Pages
  2. caspio-proxy → Editar código
  3. Reemplaza el código → Implementar
Para cambiar credenciales Caspio:
  1. caspio-proxy → Configuración
  2. Variables y secretos → Editar
  3. CASPIO_CLIENT_ID y CASPIO_CLIENT_SECRET
  4. Tipo: Secreto (nunca Texto)

Caspio y la
base de datos.

Acceso a Caspio

>threevillas.caspio.com · Panel de gestión
>API: c1abn432.caspio.com
>Perfil API: Github_manuales
>Ruta: ui/access → Web services profiles
Para añadir/editar datos de una propiedad:
  1. threevillas.caspio.com → Tables → TaVillas
  2. Buscar por Hostawayid y editar
  3. Los cambios se reflejan inmediatamente
Para regenerar el Client Secret:
  1. ui/access → Web services profiles → Github_manuales
  2. Regenerate → copiar el nuevo valor
  3. Actualizar en Cloudflare (ver pág. 4)

solución de problemas.

Test desde consola

Abre DevTools (F12) → Console en la app y pega:

fetch('https://caspio-proxy.jordi-89b.workers.dev/?action=data&table=TaVillas&id=92737')
.then(r=>r.json())
.then(d=>console.log(d))

Si devuelve {"Result":[{"Name":"Bini Chic"...}]} → todo funciona.

Seguridad y
referencia.

Estado de seguridad

✓ Credenciales Caspio
Client ID y Client Secret guardados como secretos cifrados en Cloudflare. No están en GitHub ni en el HTML público.
✓ Repositorio público
El index.html es inspeccionable por cualquiera pero solo contiene la URL del Worker, no credenciales.
→ Acción recomendada
Regenerar el Client Secret en Caspio ya que estuvo expuesto en versiones antiguas del código. Ver pág. 5.

Lo que sí es visible

>URL del Worker: caspio-proxy.jordi-89b.workers.dev
>Nombre de tabla: TaVillas
>Endpoint Caspio: c1abn432.caspio.com

Ninguno de estos datos permite acceder a los datos sin las credenciales.
